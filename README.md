# AI CMO — 퍼포먼스 마케팅 통합 분석 대시보드 (데모)

> **▶ 라이브 데모:** https://marketing-dashboard-demo-br6gpdt4i2yrshdygxanp8.streamlit.app/
> PC가 꺼져 있어도 24시간 접속됩니다. 필터·단계별 분석·탭을 직접 만져볼 수 있습니다.

Claude Code로 **직접 설계·개발**한 실제 광고 운영 대시보드의 데모 버전입니다.
표시되는 모든 수치는 **가상(임시) 데이터**이며 실제 광고주 데이터가 아닙니다. API 연동 없이 메모리에서 데이터를 생성합니다.

---

## 🎯 왜 만들었나 — 이 대시보드의 진짜 목적

다채널을 효율적으로 운영하는 것도 이유지만, **그보다 더 큰 목적은 따로 있습니다.**

> **팀원 누구나 광고 운영 현황을 한눈에 파악하고, 챗봇으로 '왜 이렇게 운영했는지' 히스토리를 묻고 기록하게 만드는 것.**

광고 운영은 매일 수많은 판단(소재 교체·캠페인 재세팅·차단·예산 조정)이 쌓이는데, 그 맥락이 한 사람 머릿속에만 있으면 팀은 매번 "이거 왜 이렇게 됐죠?"를 반복합니다. 그래서:

- **현황은 한 화면에** — 4채널 성과를 매체별로 따로 들어가지 않고 한눈에. 비전문가 팀원도 30초 안에 "오늘 광고가 어떻게 돌고 있나"를 안다.
- **히스토리는 챗봇에** — 운영 변경·판단을 자연어로 기록하고, 나중에 "그때 왜 껐지?"를 자연어로 즉시 검색. 기록과 조회를 한 곳에서.
- **그래서 — 출근하면 이 대시보드로 광고 성과를 전반적으로 파악하는 것이 '일과의 시작'이 되도록 설계했습니다.** 커피 한 잔과 함께 어제~오늘 흐름을 훑고, 이상 신호가 보이면 그 자리에서 단계별로 파고들어 원인을 캐는 흐름.

즉 이건 '분석 도구'이기 이전에 **팀의 광고 운영을 투명하게 공유하고 의사결정 히스토리를 자산으로 쌓는 협업 허브**입니다.

## 🗂 히스토리 챗봇 — 이 대시보드의 심장

> 사이드바·각 탭에 항상 떠 있는 핵심 기능. (데모에서는 가상 변경이력 기반으로 동일 UI 체험 가능)

- **기록** — 팀원(김마케터·이퍼포먼스·박CMO·최운영 등)이 "오늘 무엇을 왜 바꿨는지"를 한 줄로 남기면 운영 히스토리에 자동 누적
- **조회** — "지난주 인천 캠페인 왜 재세팅했어?" 같은 질문을 자연어로 던지면 그동안 쌓인 변경이력·판단 근거를 찾아 답변
- **목적** — 의사결정 맥락이 개인이 아닌 **팀의 공유 자산**이 되게. 담당자가 바뀌어도 히스토리가 끊기지 않게.

## 🗂 탭 구성 — 각 탭을 왜 만들었나

| 탭 | 무엇을 보나 | 왜 이 탭을 만들었나 (의도) |
|----|------------|---------------------------|
| 📊 **실시간** | 4채널 통합 KPI(전환·소진·CPM·CPC·CVR·CPA) | **일과의 시작점.** 출근 직후 "오늘 광고 전체가 어떤 상태인가"를 팀 누구나 한 화면에서 즉시 파악 |
| 📅 **일자별** | 캠페인 → 광고그룹 → 소재 단계별 분석, 기간 A/B 비교 | 실시간 탭에서 이상 신호가 보이면 **"언제부터, 어디서 나빠졌나"를 단계별로 파고들어 원인 분해** |
| 🔬 **A/B테스트** | 가설 단위 실험 관리 + 성과 + 평가/연장 | 변경을 '감'이 아니라 **가설→검증으로 기록**해, 어떤 액션이 효과였는지 팀이 공유·재현 |
| 🆕 **신규소재** | 신규 자산 투입 후 D+4 성과 추적(매체별) | 신규 소재를 **유지할지 교체할지 판단 타이밍을 규칙화** — 성급한 OFF/방치 둘 다 방지 |
| 🔁 **월간 중복DB** | DB 중복률 추이 | 같은 고객 재유입률로 **풀 소진(잠재고객 고갈)·폼 차단 효과를 월 단위로 선행 모니터링** |
| 📺 **노출 YouTube 채널** | 디멘드젠 캠페인별 노출 채널 풀 | ML이 어떤 채널 풀에 도달하는지 가시화 + **캠페인 간 채널 중복(비용 누수) 정량화** |
| 🔍 **경쟁사 분석** | LP 변경 감지(콘텐츠 해시) + 가격·훅·CTA 비교 | 경쟁사 LP 변화를 **사람이 매번 들여다보지 않아도 자동 감지** |
| 📊 **코호트** | 캠페인 수명 진단(CPA·중복DB율·CVR·노출 + 3지표 겹쳐보기) | 캠페인을 '투입 후 경과 주차'로 정렬해 **풀이 마르는 시점을 포착 → 재세팅 타이밍 근거** |
| 🎯 **KPI** | 목표 대비 현황 | 매체·캠페인 성과를 **목표 기준으로 한눈에** 평가 |

## ⭐ STAR

### S — Situation (상황)
치과 임플란트 퍼포먼스 마케팅을 **Google Ads(메인·서브)·Meta·TikTok 4개 매체**로 운영했습니다.
- 매체마다 따로 접속해 데이터를 가공하느라 시간이 오래 걸렸고, **운영 현황과 의사결정 맥락이 한 사람에게만 쌓여** 팀이 공유하기 어려웠습니다.
- **캠페인 → 광고그룹 → 소재**까지 단계별로 파고드는 분석을 누구나 같은 방식으로 할 도구가 필요했습니다.
- "왜 이 소재를 껐지?", "이 캠페인 언제 재세팅했지?" 같은 **운영 히스토리가 기록되지 않고 휘발**됐습니다.

### T — Task (과제)
- 4채널 현황을 **한 화면에서 누구나 파악**하게 만들고,
- 운영 변경·판단을 **챗봇으로 기록·조회**해 팀의 공유 자산으로 쌓으며,
- 캠페인 수명·풀 소진 시점을 데이터로 포착해 **재세팅·소재 교체·차단 타이밍을 룰 기반으로 판단**하는 시스템을 직접 구축한다.

### A — Action (실행)
Claude Code를 활용해 **대시보드 + 히스토리 챗봇 + 자동 모니터링 봇 + 경쟁사 분석 + AI 영상 파이프라인**을 직접 개발했습니다. (탭별 의도는 위 표 참고)

확립한 분석 방법론(데모에 그대로 반영):
- **코호트 분석으로 캠페인 수명 측정** — 가로축을 '투입 후 경과 주차'로 정렬해 풀이 마르는 시점을 포착
- **무너지는 인과 사슬 추적** — `CVR↓ → 중복DB율↑ → CPA↑` 순서를 한 차트에서 확인해 풀 소진을 조기 진단
- **실 전환 기준 CPA** — API 전환값이 아닌 실제 시트 전환 기준으로 효율 판정 (API는 누락 발생)
- **보수적 OFF 룰** — 단일 일자 스파이크가 아닌 7일 평균 + 3일 연속 추세로 판단해 오판 방지

### R — Result (결과)
- **흩어진 4채널을 단일 대시보드로 통합** — 출근 후 성과 파악이 일과의 시작이 되고, 매일 수작업 집계를 자동화
- **운영 히스토리를 팀 공유 자산으로** — 의사결정 맥락이 챗봇에 누적돼, 담당자가 바뀌어도 "왜 그렇게 했는지"가 끊기지 않음
- **운영 판단을 직관에서 데이터로 전환** — 캠페인 수명·풀 소진 시점을 코호트로 가시화해 재세팅/차단 타이밍을 근거 기반으로 결정

> 📌 정량 성과(CPA 개선폭·내원 전환율 등)는 검증된 실측치로 채워 넣습니다.
> `[CPA __% 개선 / 월 __건 전환 / ROAS __ 달성]`
